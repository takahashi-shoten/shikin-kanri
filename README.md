# 高橋商店 資金管理アプリ（shikin-kanri）

現場職人・小規模事業者向けの資金管理PWA。現金・銀行口座の流れをスマホで毎日記録し、
「今いくら事業資金があるか」をすぐ確認できます。音声入力で10秒記録が目標です。

- React + Vite + TypeScript / PWA（vite-plugin-pwa）
- データは端末内の localStorage に保存（サーバー不要）
- 音声入力は Web Speech API（**HTTPS環境のみ動作**。GitHub Pages や localhost はOK、`file://` は不可）

---

## 開発・起動

```bash
npm install
npm run dev      # 開発サーバー（http://localhost:5173/shikin-kanri/）
npm run build    # 本番ビルド（dist/）
npm run preview  # ビルド結果をプレビュー
```

> **音声入力について**：Web Speech API は HTTPS でのみ動作します。
> `npm run dev` の localhost は許可されますが、ビルドした `index.html` を直接ダブルクリック（`file://`）すると音声は使えません。

## 画面

| 画面 | 内容 |
|------|------|
| 🎤 音声入力 | 話すだけで日付・区分・金額・メモを自動解析。保存前に修正可 |
| ✏️ 手入力 | 音声が失敗したときのフォールバック |
| 📋 一覧 | 月別表示。入金は緑・支出は赤。タップで編集・削除（削除は確認あり） |
| 📊 集計 | 当月の支出内訳・銀行引出・売上入金の合計 |
| 💰 資金状況 | 手元現金と各銀行残高、事業資金合計、直近の銀行引出を表示 |
| ⚙️ 設定 | 初期残高入力（音声可）／CSV・JSONエクスポート／JSONインポート |

---

## 残高の考え方

残高は「初期残高 + 取引履歴の累積」で**常に再計算**されます（直接書き換えない）。

- 銀行引出：その銀行 −金額、手元現金 +金額
- 銀行入金（売上振込など）：その銀行 +金額、手元現金は変化なし

---

## 音声入力の例

- 「ホーマックで材料費1万2500円」→ 材料費 / 12,500円 / メモ:ホーマック
- 「北洋銀行から10万円下ろした」→ 銀行引出 / 北洋銀行 / 100,000円
- 「昨日ガソリン5千円入れた」→ 燃料代 / 5,000円（日付は前日）
- 設定画面で「325万円」と話すと残高欄に自動入力

## GitHub Pages 公開手順

`vite.config.ts` の `base` はリポジトリ名 `/shikin-kanri/` を想定しています。

```bash
# 1. ビルド
npm run build

# 2. gh-pages を導入（初回のみ）
npm install -D gh-pages

# 3. デプロイ（package.json の "deploy": "gh-pages -d dist" を使用）
npm run deploy
```

公開後、リポジトリの Settings → Pages で `gh-pages` ブランチが公開対象になっていることを確認してください。
URL は `https://<ユーザー名>.github.io/shikin-kanri/` です。
